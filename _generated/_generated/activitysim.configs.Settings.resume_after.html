activitysim.configs.Settings.resume_after¶

Settings.resume_after: str¶

Resume execution after this component.

There are two ways to use this setting:

  • Set to the name of a model component, to resume from after that component. This presumes that a suitable checkpoint is available.

  • Set this to a single underscore ‘_’ to resume after the last successful checkpoint of any component.
